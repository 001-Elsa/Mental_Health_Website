## 四个版本目标

### V1：让项目不玩具

- 中文源码和界面统一使用 UTF-8。
- 手机验证码注册、密码登录、JWT 会话和当前用户接口完整可用。
- DeepSeek 可用时返回真实模型回复，缺少密钥或供应商异常时使用安全降级回复。
- 情绪记录提供历史曲线、统计与趋势基线。
- 文章和社区具备基础 CRUD、评论/回复与权限校验。
- Docker Compose 编排 PostgreSQL、Redis、FastAPI 和 Nginx 前端。

### V2：让项目像真实产品

- 管理后台形成风险干预、举报、内容审核、用户、文章和规则管理闭环。
- 风险事件包含等级、分数、规则证据、模型复核信息、处置人和处置状态。
- 用户画像持续维护主要情绪、压力来源和支持偏好。
- 后台提供日活、周活、平均情绪与 14 日运营趋势。

### V3：让项目像大厂实习项目

- Redis 缓存验证码、分析结果、文章列表/详情/热门数据，并提供内存降级。
- 普通用户与管理员使用 RBAC 隔离；写接口忽略客户端伪造的 `user_id`。
- API 使用统一错误结构、`X-Request-ID`、结构化访问日志和 Pydantic 参数校验。
- Alembic 当前迁移头为 `0005_case_workflow`，已验证空库连续升级。
- GitHub Actions 执行后端测试、前端测试与生产构建。
- 性能结果见 `docs/PERFORMANCE.md`，接口契约见 `/docs` 和 `docs/API.md`。

### V4：让简历有亮点

- RAG 从审核知识文档和已发布文章中检索，输出来源引用并在证据不足时拒答。
- 对话使用“最近消息 + 长期摘要”控制上下文长度。
- 情绪趋势预测提供样本量、置信度、上下界和免责声明。
- 推荐系统以画像与近期情绪为特征，输出文章和练习及推荐原因。
- 内容安全模块覆盖规则过滤、状态机、举报处置、可见性和对象级权限。

## 简历表述

- 基于 FastAPI、React、PostgreSQL 与 Redis 设计高校心理支持平台，覆盖 AI 倾听、情绪追踪、社区治理和运营分析完整业务闭环。
- 基于 DeepSeek API 实现心理咨询多轮对话服务，设计历史上下文裁剪、长期摘要、用户画像、危机关键词识别与安全降级策略。
- 设计规则引擎与大模型二次复核相结合的风险识别模块，对高风险会话和连续情绪下降进行标记、降级回复和干预任务流转。
- 实现 RBAC 运营后台与内容安全状态机，支持 DAU/WAU、情绪趋势、高风险队列、举报审核、文章状态及敏感词管理。
- 引入 Redis 缓存、Alembic 迁移、统一异常与请求追踪、Docker Compose、GitHub Actions 和自动化测试，并完成基础压测。
